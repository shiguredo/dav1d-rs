# Decoder に flush メソッドを追加する

Model: Opus 4.6

## 概要

dav1d C API には `dav1d_flush()` 関数があるが、現在の Rust ラッパーにはこれに対応するメソッドがない。
`dav1d_flush()` はデコーダーの内部状態をリセットし、新しい位置からのデータ受付を可能にする関数である。

## 追加すべきメソッド

```rust
impl Decoder {
    /// デコーダーの内部状態をリセットする
    ///
    /// ストリーム内でシークした後に呼び出すことで、
    /// デコーダーを新しい位置からのデコードに備えさせる。
    /// 未消費のデータやバッファ中のフレームは全て破棄される。
    pub fn flush(&mut self) {
        unsafe {
            sys::dav1d_flush(self.ctx);
        }
    }
}
```

## 根拠

映像プレイヤーやストリーミングアプリケーションでは、ユーザーがシークバーを操作した際にデコーダーをリセットする必要がある。
現在の API ではデコーダーを `drop` して再生成するしか方法がなく、コンテキストの再初期化コストが無駄にかかる。
`dav1d_flush()` を使えば既存のコンテキストを維持したまま状態だけをリセットできる。

また、`finish()` メソッドは現在何もしていないが、これはストリーム末尾の通知用であり、
シーク時のリセットとは意味が異なる。両者は別のメソッドとして存在すべきである。

## 解決方法

`Decoder::flush()` メソッドを追加し、内部で `dav1d_flush()` を呼び出すようにした。
