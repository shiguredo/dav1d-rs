# dav1d のランタイムバージョン取得関数を追加する

Model: Opus 4.6

## 概要

dav1d C API には `dav1d_version()` (文字列) と `dav1d_version_api()` (数値) という
ランタイムバージョン取得関数があるが、現在の Rust ラッパーではこれらが公開されていない。

既存の `BUILD_VERSION` 定数はビルド時に Cargo.toml から読み取った値であり、
実際にリンクされている dav1d ライブラリのバージョンとは異なる可能性がある。

## 追加すべき関数

```rust
/// リンクされている dav1d ライブラリのバージョン文字列を返す
pub fn version() -> &'static str { ... }

/// リンクされている dav1d ライブラリの API バージョンを数値で返す
///
/// フォーマット: ((major << 16) | (minor << 8) | patch)
pub fn version_api() -> u32 { ... }
```

## 根拠

ビルド時のバージョン (`BUILD_VERSION`) とランタイムのバージョンが一致するかどうかを
アプリケーション側で確認する手段が必要である。
特にシステムの共有ライブラリとリンクしている場合、
ビルド時と異なるバージョンの dav1d がロードされる可能性がある。
診断ログやバグレポートにランタイムバージョンを含めることで、
問題の切り分けが容易になる。

## 解決方法

`version()` と `version_api()` 関数を追加し、それぞれ `dav1d_version()` と `dav1d_version_api()` を呼び出すようにした。
